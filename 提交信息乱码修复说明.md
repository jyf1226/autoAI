# 提交信息乱码修复说明

## 问题现象

- 在 PowerShell 中通过管道将中文提交信息传给 `git commit -F -` 时，提交主题出现 `????`。
- 结果是“中文在前 / 英文在后”的格式被破坏（中文不可读）。

## 根因

- 当前会话编码链路不是稳定 UTF-8（命令行输入、管道、Git 接收端存在编码不一致）。
- 中文字符在传输过程中被降级为 `?`。

## 稳定解决方案（已落地）

新增脚本：

- `scripts/commit-bilingual.ps1`

做法：

1. 在脚本中显式设置 UTF-8 编码：
   - `Console.InputEncoding`
   - `Console.OutputEncoding`
   - `$OutputEncoding`
2. 先把双语提交信息写入 UTF-8 临时文件。
3. 用 `git commit -F <tempfile>` 提交，避免管道编码损坏。

## 使用方式

```powershell
.\scripts\commit-bilingual.ps1 `
  -SubjectZh "中文主题" `
  -SubjectEn "English subject" `
  -BodyZh "中文正文（why）" `
  -BodyEn "English body (why)"
```

## 约定

- 提交主题统一：`中文在前 / English after`
- 正文统一：先中文，再英文，语义保持一致
